# 开发日志

## 2026-04-19 项目初始化与核心功能

### 需求背景
大学生求职季需要同时管理多个公司的申请，包括截止日期、面试进度、材料状态等。现有工具（Excel、备忘录）不够直观，需要一个 Trello 风格的看板来可视化管理。

### 技术选型
- React 18 + TypeScript + Vite：类型安全，开发体验好
- @dnd-kit：React 生态最现代的拖拽库，轻量且支持无障碍
- CSS Modules：样式隔离，不需要引入 Tailwind 等额外依赖
- LocalStorage：个人工具不需要后端，浏览器存储足够

### 完成功能
- 6 列看板 + 暂存区，2×3 网格布局
- @dnd-kit 拖拽移动卡片
- 新增/编辑/删除申请（Modal 表单）
- 粘贴招聘文本智能识别
- 公司名 combobox（可输入可下拉）
- 截止日期到期提醒
- 搜索过滤 + 数据统计
- LocalStorage 持久化

## 踩坑记录

### 1. 布局从横向滚动改为 2×3 网格
**问题**：最初用 `display: flex` 横向排列 6 列，屏幕上留白严重，每列太窄。
**解决**：改为 `display: grid; grid-template-columns: repeat(3, 1fr); grid-template-rows: 1fr 1fr`，同时去掉 Column 的 `flex: 0 0 280px` 固定宽度，让列自适应网格。

### 2. 暂存区的设计
**问题**：批量录入时，用户需要先把卡片统一放到一个地方，之后再分类拖拽。最初没有这个功能。
**解决**：在右侧增加独立的"暂存区"列，用不同背景色（淡紫色 + 虚线边框）与左侧状态列区分。新增申请默认进入暂存区。

### 3. 材料清单功能被砍掉
**问题**：最初设计了材料提交清单（简历、成绩单等），但实际使用中发现这个功能意义不大，反而增加了录入负担。
**解决**：移除材料清单，改为"工作内容"和"任职要求"两个文本字段，更贴合实际需求。

### 4. 薪资字段被砍掉
**问题**：招聘文本中薪资格式不统一，且对求职管理来说不是核心信息。
**解决**：直接移除薪资字段，简化表单。

### 5. 链接导入方案的演变
**问题**：最初想通过 URL 直接抓取招聘页面信息，但纯前端有 CORS 限制，免费代理不稳定。
**尝试**：先做了 CORS 代理方案 → 发现大部分招聘网站屏蔽 → 改为粘贴文本智能识别。
**最终方案**：用户复制招聘页面文字，粘贴到文本框，正则提取各字段。

### 6. 粘贴识别的正则踩坑（最大的坑）

#### 6.1 城市名被识别为公司名
**问题**：解析逻辑按顺序取前几行非空行，城市名（如"杭州"）被误识别为公司名。
**解决**：维护一个城市名正则列表，匹配到的行跳过，同时单独提取为"工作地点"字段。

#### 6.2 面试地点 vs 工作地点
**问题**：文本中同时有"面试地点：远程"和"工作地点：上海、杭州"，"远程"被填入了工作地点。
**解决**：只匹配 `工作地点：` 标签，不匹配 `面试地点：`。

#### 6.3 句中的"岗位要求"被误识别为段落标题
**问题**：字节的招聘文本里有"为符合岗位要求的同学提供转正机会"，其中"岗位要求"被正则匹配为段落标题，导致切分位置全错。
**解决**：段落标题正则加 `^` 行首锚定 + `m` multiline flag，只匹配独立成行的标题。

#### 6.4 "岗位基本要求"等变体无法识别
**问题**：美团的文本用的是"岗位基本要求"而不是"任职要求"，正则没覆盖到。
**解决**：扩展正则，加入"岗位基本要求"、"具备以下条件优先"等变体。

#### 6.5 职位描述中混杂团队介绍
**问题**：字节的"职位描述"段落里先是 ByteIntern 说明和团队介绍，然后才是编号条目（1、2、3...），导致工作内容被团队介绍文字污染。
**解决**：从段落中提取编号条目（`/^[0-9]+[、.）)]/`），如果有 2 条以上就只保留编号条目，过滤掉非结构化文字。

#### 6.6 解析架构从正则改为段落切分
**问题**：最初用单个正则匹配整段内容，但不同平台格式差异大，正则的 lookahead 容易互相干扰。
**解决**：改为先找所有段落标题的位置，按位置切分文本，每个段落独立提取。这样各段落互不影响。

### 7. 粘贴识别入口不够醒目
**问题**：最初"粘贴识别"是右上角一个小按钮，用户容易忽略。
**解决**：新增申请时，粘贴区域默认展开在表单最顶部，一眼就能看到。

### 8. 卡片上工作内容显示太少
**问题**：最初只显示前 60 个字符的纯文本，信息密度低。
**解决**：改为按条目显示，每条截取前 25 字，最多显示 3 条，用列表样式呈现。

### 9. 卡片上链接不可点击
**问题**：链接只是一个 emoji 图标，没有交互。
**解决**：改为 `<a>` 标签，点击在新标签页打开。同时用 `onPointerDown` + `onClick` 的 `stopPropagation` 防止触发拖拽。
